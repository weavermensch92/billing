# `/gz-conflict` — Conflict 수동 기록

## 목적
자동 감지 외에 사용자가 명시적으로 규칙 충돌 상황을 기록.

## 트리거
- **수동**: 사용자가 `/gz-conflict` 입력
- **자동**: `99_protocol § 1` 4조건 중 하나 감지 시 (반복 버그 3회 / 규칙 충돌 / 합의 미달 3회 / 정합성 위반)

## 입력
```
어떤 규칙(들)이 충돌? (예: G-052 vs PW-007)
어떤 상황에서?
현재 작업 어디까지 진행?
```

## 실행
`99_protocol.md § 1` 그대로. 세부:

```
1. 4조건 중 어떤 것인지 자동 판정 (수동 호출이면 사용자가 선택)
2. 현재 세션 상태 스냅샷 (chain, loaded files, 관련 규칙 ID)
3. .context/issue/conflicts-YYYY-MM-DD-HH-MM_<slug>.md 생성
4. 체인 일시 중단 (파일만 기록, 실행 중단은 해당 체인 규칙 따름)
5. 해결 제안 3가지:
   - 우선순위 확정 (규칙 A가 B보다 우선)
   - 조건부 적용 (특정 상황에서만 A)
   - 신규 규칙으로 병합
6. 사용자 선택 후 해당 본문 파일 수정 제안
7. /gz-send-issue 로 GitHub Issue 생성 권고
```

## 출력
Conflict 파일 경로 + 세션 상태 + 해결 옵션 3가지.

## 예시
```
/gz-conflict

"어떤 규칙들이 충돌하나요?"
사용자: "G-052 서버 필터링 우선 vs PW-007 클라이언트 렌더링 제어"

"어떤 상황?"
사용자: "칸반 HITL 필터에서 실시간 업데이트 우선순위"

[파일 생성]
  .context/issue/conflicts-2026-04-18-14-30_kanban-hitl-filter-routing.md

[해결 옵션]
  [1] G-052 우선 (서버 필터링 → 재쿼리)
  [2] PW-007 우선 (클라이언트 캐시 → 낙관적 업데이트)
  [3] 조건부 (개수 ≤ 50: PW-007 / > 50: G-052)
```

## 금기
- Conflict 기록 없이 우회 판단 금지 (기록이 지식 축적의 시작)
- 다른 세션의 Conflict 파일 자동 병합 금지 (사용자 검토 필수)
- 해결 제안 없이 파일만 생성하고 종료 금지
