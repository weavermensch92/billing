# `/gz-pattern` — 패턴 승격 후보

## 목적
동일한 수정·결정·코드 블록이 3회 이상 반복되면 규칙 승격 후보로 제안. Knowledge로 기록 후 팀 리뷰 경유 규칙 승격.

## 트리거
- **자동**: Claude Code가 세션 중 반복 패턴 3회 이상 감지 시 자동 제안
- **수동**: `/gz-pattern` 입력 시 현재까지의 패턴 후보 목록 출력

## 입력
없음 (자동) 또는 특정 패턴 slug 지정.

## 실행
```
1. 세션 히스토리 스캔 (최근 50개 턴)
2. 동일 수정/결정/코드 블록 빈도 카운트
3. 빈도 3+ 감지 시 패턴 후보 생성
4. 기존 규칙 파일에 동일 규칙 있는지 중복 스캔 (00_index 검색)
5. 승격 대상 파일 + 신규 ID 제안
6. 사용자에게 4지선다 제시: [승격] [수정 후 승격] [기각] [보류]
```

## 출력
```
💡 패턴 감지 (N회 반복)
   내용: <요약>
   첫 발견: <턴 번호 또는 시각>
   승격 대상 파일: <예상 파일>
   승격 후보 ID: <신규 ID 제안>
   기존 중복: <있음/없음>

   [규칙으로 승격] [수정 후 승격] [기각] [보류]
```

### 승격 시
```
1. .context/issue/knowledge-YYYY-MM-DD-HH-MM_<pattern-slug>.md 생성
2. 00_index.md 업데이트 제안 (신규 ID 할당)
3. 본문 파일에 규칙 추가 안내 (사용자가 직접 수정)
4. /gz-send-issue 로 GitHub Issue 생성 제안 (팀 리뷰용)
```

## 예시
```
Claude Code (자동):
💡 패턴 감지 (3회 반복)
   내용: "TypeScript 파일 생성 시 파일명을 camelCase로 작성 → kebab-case로 수정"
   첫 발견: 세션 시작 후 12분
   승격 대상 파일: rules/07_coding_standard.md (G-122에 이미 존재 ✓)
   신규 승격 불필요 — 기존 G-122 강화 제안

   [G-122 강화] [별도 신규 규칙] [기각] [보류]
```

## 금기
- 2회 반복으로 패턴 승격 제안 금지 (최소 3회)
- 기존 규칙과 중복 체크 없이 신규 ID 발급 금지
- 사용자 승인 없이 본문 파일 직접 수정 금지 (Knowledge 파일만 자동 생성 OK)
