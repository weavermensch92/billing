# `/gz-spec <ID>` — 규칙 단건 조회

## 목적
규칙 ID 하나를 입력받아 본문 + 연관 규칙을 출력.

## 트리거
- **수동**: 사용자가 `/gz-spec <ID>` 입력
- **자동**: 체인 실행 중 규칙 본문 확인 필요 시 내부 호출

## 입력
규칙 ID 하나. 허용 접두어:
`G-xxx` / `D-xxx` / `PA-xxx` / `PL-xxx` / `PW-xxx` / `I-xxx` / `L-xxx` / `H-xxx` / `F-xxx`

## 실행
```
1. ID 접두어로 네임스페이스 판별
2. 00_index.md 에서 파일 경로 조회
3. 해당 파일이 존재하면 로드 → 본문 추출
4. 본문 파일 부재 시:
   - 00_index 의 요약만 반환 + "본문 미검증" 경고 (90 § Fallback)
5. 연관 규칙 (relatedRules / 참조 섹션) 목록 병행 출력
```

## 출력
```
ID: <규칙 ID>
파일: <경로>
강제: MUST / SHOULD / MAY
본문: <요약 또는 전문>
연관 규칙: <ID 리스트>
```

## 예시
```
/gz-spec D-025

→ D-025 동시성 제어 결정
  파일: products/lucapus/rules/spec_db_persistence.md
  강제: MUST

  본문:
  "동시 접근 가능한 자원의 상태 변경 시 락 전략을 명시한다.
   - 비관적 락: DB 수준 잠금. 안전하나 TPS 저하.
   - 낙관적 락: 버전 충돌 시 재시도. 빠르나 충돌 빈발 시 UX 저하.
   - 선택 기준: 충돌 빈도 × 트랜잭션 크기."

  연관 규칙:
  - D-072 이벤트 발행 순서
  - D-080 트랜잭션 경계
```

## 금기
- ID 없이 실행 금지 (필수 인자)
- 다중 ID 동시 조회 금지 (각 ID 별도 실행)
- 본문 없을 때 추측 생성 금지 (Fallback 경고만)

## 참조
- `00_index.md § 0~9` 네임스페이스 + ID 카탈로그
- `90_execution_chain § Fallback` 본문 부재 시 동작
