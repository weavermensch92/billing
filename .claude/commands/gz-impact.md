# `/gz-impact <변경대상>` — 제품 간 임팩트 분석

## 목적
Gridge AIMSP의 제품 간 연동(AiOPS / LucaPus / Wiring)을 고려한 변경 영향 분석. 그릿지 전용 명령어.

## 트리거
- **수동**: `/gz-impact <변경 설명>` 입력
- **자동**: S 체인 / M 체인 진입 시 다른 제품 연동점이 있다고 판단되면 권고

## 입력
변경 대상 자연어 설명. 예: "AiOPS 로그 스키마에 tenant_id 추가"

## 실행
```
1. 변경 대상이 속한 제품 파악 (접두어 ID / 키워드로)
2. integrations/*.md 전체 스캔 (I-xxx, L-xxx, H-xxx ID)
3. 영향 받는 다른 제품 목록 + 깨질 수 있는 연동점 식별
4. 양쪽 제품 규칙 충돌 검토 (product 라우터의 "전용 절대 규칙" 교차 검증)
5. Mode 영향 분석 (Mode B 고객 마이그레이션 필요 여부 등)
6. 위계 영향 분석 (L3 이상 접근 필요 / OA 승인 필요 여부)
7. 권장 실행 순서 출력
```

## 출력
```
영향 분석:
  [원천] <제품> <ID>
  [연쇄]
    - <I-xxx> 연동 이름: 영향 설명
  [Mode 영향]: ...
  [위계 영향]: ...

권장 순서:
  1. ...
  2. ...
```

## 예시
```
/gz-impact "AiOPS 로그 스키마에 tenant_id 추가"

→ 영향 분석:
  [원천] AiOPS PA-001
  [연쇄]
    - I-001 AiOPS → Wiring 로그 파이프라인: Wiring 보고서 탭 영향
    - I-003 LucaPus → AiOPS 에이전트 호출 로깅: LucaPus 호출 페이로드 수정 필요
  [Mode 영향]
    - Mode B 온프레: 고객 DB 마이그레이션 필요 → 조직 승인 선행
  [위계 영향]
    - L3 기술 리드 이상 접근 필요 (멀티테넌시 설정)

  권장 순서:
  1. Wiring 보고서 탭 하위 호환성 확인
  2. LucaPus 호출부 업데이트
  3. AiOPS 스키마 마이그레이션
  4. Mode B 고객 별도 공지
```

## 금기
- integrations/ 스캔 없이 결과 반환 금지 (실제 연동점 확인 없이 추측 금지)
- "영향 없음" 단정 금지 (최소한 "확인 범위 내 영향 없음" 표현)
- 외부 파트너/고객에게 이 출력 그대로 공유 금지 (내부 용어 포함)
