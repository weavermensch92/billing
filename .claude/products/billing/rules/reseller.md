# Billing / Rules / Reseller — 리셀러 구조 원칙

> **PB-001** — 본 서비스의 법적 정체성. 자금 흐름이 PG가 아닌 재판매(resale) 임을 정의하고, 전자금융거래법 경계를 넘지 않도록 시스템·문서·용어를 강제한다.

---

## PB-001-01. 자금 흐름 (절대 규칙)

```
[그릿지 법인카드 한도]
     │ 그릿지 자기 자금 (대지급)
     ▼
[VCN 으로 AI 서비스 결제]
     │ 그릿지 명의로 지불
     ▼
[AI 벤더]
     │
     ↓ 월말 청구서 발행
[그릿지 → 고객사에 재청구]
     │
     ↓ 고객사가 그릿지에 입금
[그릿지 계좌 수납 확인]
```

**절대 금지**: 고객 돈 → 그릿지 경유 → 가맹점 경로 생성.

위반 시 전자지급결제대행업(PG) 등록 의무 발생. 스타트업에 사실상 불가능 (자본금 3억~20억, 전문인력 5인, 금융위 등록).

## PB-001-02. 법적 해석 근거

한국 전자금융거래법 상 "전자적 방법으로 재화 구입·용역 이용 대가 정산을 대행하거나 매개" → PG 등록 필요.

본 서비스는 다음 조건을 만족하여 재판매(resale)로 해석:
- 고객 자금이 그릿지 계좌를 **경유하지 않음**
- 그릿지가 **자기 돈으로 먼저 결제** (대지급)
- 이후 고객에게 **재판매 매출로 청구**
- 부가세 흐름: 그릿지 매입 → 그릿지 매출 (표준 B2B 재판매 회계)

2026년 12월 17일 시행 개정 전자금융거래법이 통신판매중개업자 등 부수적 정산 대행 전금법 배제 방향 → 본 구조 해석 명확화 전망.

**단, 본격 영업 전 1회 법무 자문 필수** (플레이북 § 9).

## PB-001-03. 용어 사용 금지 (대내·외 문서·계약서·UI·코드)

"결제 대행" 및 PG 를 연상시키는 언어 전면 회피. 이는 **G-004 외부 노출 금지어**에 Billing 제품 전용으로 추가.

| ✅ 권장 | ❌ 회피 |
|---|---|
| AI 서비스 통합 관리 | AI 결제 대행 |
| AI 계정 MSP | 결제 대행 서비스 |
| 그릿지가 AI 서비스를 재판매 | 그릿지가 고객 결제를 대행 |
| 상품·용역 매출 세금계산서 | 정산서·결제대행수수료 |
| 대지급·재청구 | 지급결제 |
| Account Manager (AM) | 결제 처리자 |

위반 패턴 (PR 게이트 T3·T4 차단 대상):
- `grep -rE "결제\s*대행|결제대행|payment\s*processing"` → 위반
- 계약서·랜딩페이지·운영 콘솔 copy 반드시 사전 검토

## PB-001-04. 데이터 모델 강제

`transactions` 테이블 설계가 재판매 구조 반영:

```sql
gridge_cost_krw       BIGINT  -- 그릿지 매입가 (벤더 실결제)
customer_charge_krw   BIGINT  -- 고객 재청구가
gridge_margin_krw     BIGINT  -- 마진 (내부 전용 필드)
is_anthropic_passthrough BOOLEAN  -- 패스스루 구분
```

- `gridge_cost` 와 `customer_charge` 분리 저장 → 매입 ≠ 매출 명시
- 모든 결제는 먼저 그릿지 명의 → 월말 묶음 재청구

절대 금지 필드:
- 고객→그릿지 이체 매개용 임시 보관 계정
- 고객 자금 에스크로 / 프리페이드 지갑 구조

## PB-001-05. 계약서 / 약관 명시

고객 계약서에 명시 필수:
- 그릿지는 AI 서비스 **재판매자** (resale)
- 매입가 + 매출가 투명 공개 (크레딧백 차감 방식)
- 대지급 금액의 신용한도 정책 (`org_contracts.monthly_credit_limit_krw`)
- 미납 시 VCN 폐기 권한 그릿지 보유
- 전자지급결제대행업 미해당 근거

## PB-001-06. 자동 검증 체크리스트

PR 생성 / 문서 저장 시 T3·T4 게이트에서 검증:

- [ ] 코드·문서·UI copy 에 "결제 대행" 포함 여부
- [ ] 새 API 엔드포인트가 고객→벤더 직접 자금 이동 경로 생성 여부
- [ ] DB 스키마에 고객 자금 보관 테이블 신설 여부
- [ ] `transactions.gridge_cost_krw` 필드 삭제 / 병합 시도 여부
- [ ] 고객 세금계산서에 "결제대행수수료" 항목 포함 여부

## PB-001-07. 포지셔닝 (대외)

**"Brex 가 아니다, 법인카드도 아니다, AI 계정 관리 MSP 다."**

1. 세금계산서 발행
2. 벤더 약관 검증
3. 전담 매니저 서비스
