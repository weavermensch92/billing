# Billing / Rules / Phase Transition — Phase 0 → 1 → 2 전환 체크포인트

> **PB-013** — 각 Phase 전환 시 반드시 확인해야 하는 체크포인트. 기술 / 운영 / 재무 3축 검증 후 진입.

---

## PB-013-01. Phase 정의 (요약)

| Phase | 기간 | 고객 규모 | 자동화 수준 |
|---|---|---|---|
| **Phase 0** | 0~2개월 | Alpha 1개사 | 수동 (카드사 포털, CSV import) |
| **Phase 1** | 2~6개월 | 2~5개사 | API 자동화 (카드사, Smart Bill, 웹훅) |
| **Phase 2** | 6개월~ | 20개사+ | 표준 상품 + ML 이상 감지 |

## PB-013-02. Phase 0 → 1 전환 체크포인트

### 기술 체크 (모두 ✅ 필수)

- [ ] **카드사 B2B API 계약** (신한 V-Card) 체결 + 샌드박스 2주 검증
- [ ] **Smart Bill API 연동** 완료 + 월 100건 자동 발행 테스트
- [ ] **오픈뱅킹 수납 매칭** 자동화 또는 수동 프로세스 표준화
- [ ] **1Password Connect 서버** 배포 + 자동 공유 링크 생성
- [ ] **백업 카드사 (KB)** 연동 + Primary/Backup 자동 전환 테스트
- [ ] **웹훅 서명 검증** (HMAC) 100% 통과
- [ ] **Staging 환경** 완비 + Alpha 데이터 복제 테스트 가능

### 운영 체크

- [ ] **Alpha 고객 3개월 안정 운영** (월말 마감 3회 오차 < 0.5%)
- [ ] **거절 대응 SLA 95%+** 달성 (최근 3개월)
- [ ] **월말 마감 공수 40시간 → 8시간** 감소 측정
- [ ] **runbook 7개** 모두 최소 1회씩 실전 적용
- [ ] **법무·세무 자문** 완료 + 이슈 없음

### 재무 체크

- [ ] **Alpha 월 매출 ₩500만 이상** 3개월 연속
- [ ] **대지급 리스크 관리** (연체 0건 또는 조치 완료)
- [ ] **Anthropic 파트너십 승인** (패스스루 활성화)
- [ ] **2번째 고객 계약** 체결 or 확정 파이프라인

### 고객 체크

- [ ] **NPS ≥ 8** (Alpha D+90)
- [ ] **업셀 시그널** 최소 1건 감지 (AiOPS 또는 Wiring)

## PB-013-03. Phase 1 → 2 전환 체크포인트

### 기술 체크

- [ ] **표준 온보딩 자동화** (계약 서명 → org 등록 → 초대 발송 자동)
- [ ] **자동 이상 감지 ML** 기반 (Phase 1 룰 + 학습)
- [ ] **다중 카드사 라우팅** (신한 + KB + 해외 VCN Phase 2 후보)
- [ ] **SSO 연동** (SAML / OIDC / SCIM, 엔터프라이즈 요건)
- [ ] **랜딩 페이지 + 자동 가입 플로우** 구현
- [ ] **티어 2 (주간) + 티어 3 (선불) 자동 실행** 검증 완료

### 운영 체크

- [ ] **고객 5개사 이상 6개월 무사고 운영**
- [ ] **운영 공수 고객당 평균 월 4시간 이하** 달성
- [ ] **CSM 1명당 10개 고객** 관리 가능한 프로세스
- [ ] **Finance 전담자** 채용 or 역할 분리 완료
- [ ] **리멤버 B2B 캠페인** 투입 준비 (계약서·영업 자료 표준화)

### 재무 체크

- [ ] **월 매출 ₩5,000만 이상** (Phase 1 말 기준)
- [ ] **매출총이익률 10% 이상** 유지 (크레딧백 종료 고객 기준)
- [ ] **Anthropic 파트너십 재협상 성공** (15% 할인 또는 커밋 계약)
- [ ] **예측 가능한 CAC** (고객당 크레딧백 소모 패턴 데이터)

### 제품 체크

- [ ] **전환율 KPI 측정 개시**:
  - Billing → AiOPS ≥ 20%
  - Billing → Wiring ≥ 10%
- [ ] **크로스셀 케이스 스터디** 최소 2건 작성

## PB-013-04. Phase 전환 금지 조건 (Red Flag)

다음 중 **하나라도** 있으면 **전환 보류**:

### Phase 0 → 1 금지
- 월말 마감 오차율 > 1%
- 거절 SLA < 80%
- 법무 자문 미해결 이슈 존재
- Alpha 고객 NPS < 6
- Anthropic 파트너십 승인 없음

### Phase 1 → 2 금지
- 6개월 내 운영 사고 1건 이상 (결제 오류, 데이터 유출 등)
- 연체율 > 5%
- 고객 해지율 > 20% (6개월 기준)
- CSM 대응 실패 케이스 누적

## PB-013-05. 전환 프로세스

### Super 주관 분기 전환 리뷰

매 분기말 (3·6·9·12월):
1. 체크포인트 자동 스캔 (`node scripts/phase-check.js`, v0.22+ 추가)
2. Red Flag 수동 검토
3. 전환 가부 결정
4. 전환 시: v0.x+1 개발 착수 + 98_governance 이력 기록
5. 보류 시: 미해결 이슈 담당자 배정 + 다음 분기 재검토

### 전환 공식 절차

```
[체크포인트 통과] → Super 서명 → 98_governance 이력 추가
     ↓
[다음 Phase 개발 착수]
     ├─ 신규 기능 구현 (카드사 API 등)
     ├─ 기존 운영 유지 (Phase 0 고객 계속 서비스)
     └─ 점진적 마이그레이션 (고객별)
     ↓
[모든 고객 새 Phase 적용 완료] → 전환 완결
```

Phase 전환 중에도 **기존 고객 서비스는 중단 없음** (점진적).

## PB-013-06. Phase 별 기술 전환 표 (요약)

| 기능 | Phase 0 | Phase 1 | Phase 2 |
|---|---|---|---|
| VCN 발급 | 카드사 포털 수동 | 카드사 API 자동 | 다중 카드사 라우팅 |
| 결제 수집 | 일일 CSV | 실시간 웹훅 | 웹훅 + 이상 트래픽 분산 |
| 티어 2 주간 | 테이블/API만 | 실행 로직 | 안정화 |
| 티어 3 선불 | 테이블/API만 | 실행 로직 | 안정화 |
| 세계서 | Smart Bill 수동 | Smart Bill API | API + 수정 발행 워크플로 |
| 오픈뱅킹 | 수동 확인 | 웹훅 자동 매칭 | 분할 매칭 |
| 환율 | Luna 수동 | 카드사 API | 환차 예측 |
| 1Password | 수동 | 1Password Connect | 고객별 전용 볼트 |
| SSO | 없음 | 없음 | Google/Okta/Azure |
| 이상 감지 | 룰 기반 | 룰 + 통계 | 룰 + ML |

## PB-013-07. 각 Phase 의 "성공 정의"

### Phase 0 성공 = Alpha 3개월 무사고
- VCN 해외결제 승인률 ≥ 95%
- 월말 정산 오차율 < 0.5%
- 첫 고객 NPS ≥ 8

### Phase 1 성공 = 자동화 + 2번째 고객
- 운영 공수 월 40시간 → 월 8시간
- 청구서 자동 발행율 100%
- 두 번째 고객 계약 체결

### Phase 2 성공 = 표준 상품화 + 확장
- 월 신규 계약 5개사
- Billing → AiOPS 전환율 ≥ 20%
- Billing → Wiring 전환율 ≥ 10%
