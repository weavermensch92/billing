# Wiring / Screens / Org Admin — 규칙 본문

> PW-013 본문. Org Admin(OA) 전용 조직 관리 페이지.
> OA 로그인 시에만 네비게이션에 "조직 관리" 탭 추가 노출.

---

## PW-013 — Org Admin 페이지 (MUST)

### 위치

- 경로: `/org`
- 접근: **OA 전용** (L1 이하 접근 시 403)
- 네비게이션 노출 조건: `session.level === 'OA'` (G-061-01)

### 섹션 구조

```
/org/dashboard        조직 대시보드 (홈)
/org/teams            팀 관리
/org/rules            조직 규칙 (MUST)
/org/infra            인프라 모드 확인 (읽기 전용)
/org/sso              SSO 설정
/org/audit            조직 감사 로그
/org/reports          조직 보고서
```

## PW-013-01 — 조직 대시보드 (MUST)

### 상단 KPI 카드

```
전체 적합화 평균  |  전체 비용  |  활성 프로젝트 수  |  HITL 대기 총합
     76%          |   모드별    |      8개           |    23건
```

### 팀별 적합화 바 차트

```tsx
type TeamSummary = {
  name: string;
  memberCount: number;
  adaptScore: number;      // 0~100
  activeProjects: number;
  costThisMonth: number;   // 모드별 분기
  status: 'on-track' | 'at-risk';
};
```

표시:
```
Platform팀 ████████░░  95%  $18.20  ● 달성
Backend팀  ██████░░░░  78%  $24.10  ● 진행중
Frontend팀 ███████░░░  65%  $19.80  ● 달성
```

팀 행 클릭 → 드릴다운 (`/org/teams/{teamId}`)

### 크로스 프로젝트 카드

전 팀 프로젝트 간 규칙 충돌 / 중복 / 미적용 감지:
```
⚠ 규칙 충돌 (1건)
  "API 재시도 정책"
  Backend팀: MUST 3회 vs Platform팀: SHOULD 미적용

⚠ 규칙 누락 (3건)
  조직 MUST "BCrypt 해싱" 미적용: 프로젝트 B, C
```

### 전체 비용 추이

주간 / 월간 선택 가능. 팀별 적층 차트. Mode 별 단위.

## PW-013-02 — 팀 관리 (MUST)

### 기능

| 액션 | 설명 | 감사 로그 |
|---|---|---|
| [+ 팀 생성] | 새 팀 생성 + 팀 규칙 초기화 | ✅ |
| 팀원 이동 | 드래그 앤 드롭 or 팀 변경 드롭다운 | ✅ |
| 팀 삭제 | 확인 모달 + "이 팀 규칙은 보관됩니다" | ✅ |
| 팀명 수정 | 인라인 편집 | ✅ |

### 팀원 역할 관리

- L1 / L2 / L3 / L4 드롭다운으로 변경
- OA 자신은 다른 OA에게만 OA 역할 부여 가능 (자가 부여 금지)
- 변경 즉시 감사 로그 (G-141) + 해당 사용자에게 이메일 알림

### SSO 역할 자동 매핑

SSO 설정(`/org/sso`)에서 지정된 매핑 있으면 수동 변경 불가 (회색 처리 + 툴팁).

## PW-013-03 — 조직 규칙 (MUST)

G-042 정합. 조직 MUST 규칙은 **모든 하위 팀/프로젝트 강제**.

### UI

```
조직 규칙 (MUST — 전 팀 강제)
├─ 🔒 BCrypt 해싱 필수
├─ 🔒 JWT 15분 만료
├─ 🔒 감사 로그 UPDATE 금지
└─ 🔒 개인정보 암호화

[+ 새 조직 규칙 추가]
```

### 추가 시 경고 모달

```
⚠ 조직 MUST 규칙 추가
  이 규칙을 추가하면 **모든 팀/프로젝트**에 즉시 강제됩니다.
  기존 프로젝트에 영향이 갈 수 있습니다.

  규칙 이름: [                          ]
  강제 수준: [MUST ▼]
  설명: [                          ]

  [취소]  [규칙 추가]
```

### 해제 불가 (MUST)

- 🔒 아이콘 표시 (자물쇠)
- 팀/프로젝트에서 해제 시도 감지 시 Conflict (G-042 위반)
- 조직 규칙 본인이 폐기해야 함 (OA만, 감사 로그 필수)

### 관련 규칙

SHOULD로 추가 시 하위에서 MUST/MAY 오버라이드 가능 — `03_hierarchy § 2` 참조.

## PW-013-04 — 인프라 모드 확인 (MUST)

**읽기 전용**. OA는 변경 불가 (슈퍼 어드민 전용, G-081).

```
현재 인프라 모드: Mode B (온프레미스)

세부:
  - AI 실행: 고객 서버 (vLLM Llama-3)
  - 결제: 고객 자체
  - 데이터 위치: 고객 인프라

변경이 필요하신가요? 슈퍼 어드민(그릿지 운영팀)에 요청하세요.
```

변경 요청 버튼은 별도 이메일 폼 링크.

## PW-013-05 — SSO 설정 (MUST)

G-046 / G-142-02 정합.

### 지원 프로토콜

- SAML 2.0 (Okta / Azure AD / OneLogin)
- OIDC (Auth0 / Keycloak / Google Workspace)
- SCIM 2.0 (사용자 프로비저닝)

### UI

```
SSO 연결 상태: ✅ 연결됨 (Okta)

프로토콜: SAML 2.0
Entity ID: https://gridge.ai/saml/metadata
Callback URL: https://wiring.gridge.ai/sso/callback

역할 자동 매핑:
  Okta Group: "Engineering Leaders"    → L3 기술 리드
  Okta Group: "Engineers"              → L4 개발자
  Okta Group: "Product Managers"       → L2 PM
  Okta Group: "Executive"              → L1
  Okta Group: "IT Administrators"      → OA

[역할 매핑 편집]  [테스트 로그인]  [연결 해제]

SCIM Token: ***...abc123  [회전]  [복사]
```

### 보안

- SAML assertion 서명 검증 필수
- Metadata URL은 HTTPS만
- Token 회전 분기별 권장

## PW-013-06 — 조직 감사 로그 (MUST)

G-141 정합. immutable.

### 표시 범위

- 전 팀 / 전 프로젝트의 감사 대상 행위 20종
- 최근 1년 기본, 보유 기간 설정 가능 (1~5년)

### UI

```
필터: [행위 ▼] [행위자 ▼] [기간 ▼] [검색: ...]

| 시각              | 행위                | 행위자     | 대상                | 이전 → 이후       |
|------------------|--------------------|-----------|--------------------|------------------|
| 2026-04-18 14:20 | 역할 변경           | OA (김영희) | 이철수             | L4 → L3          |
| 2026-04-18 13:05 | 조직 MUST 추가      | OA (김영희) | 규칙 "JWT 만료 15분" | (신규)           |
| 2026-04-18 11:42 | 인프라 모드 변경     | 슈퍼어드민  | 전체                | A → B            |

[CSV 다운로드]  [PDF 다운로드]  [전체 ZIP 내보내기]
```

### 검색 / 필터

- 20종 행위별 드롭다운
- 행위자 (이름 / 이메일)
- 대상 (프로젝트 / 팀 / 사용자)
- 기간 (날짜 범위 피커)

### Immutable 원칙

- UPDATE / DELETE 버튼 **존재 금지**
- 다운로드 시 무결성 해시 포함 (SHA-256)

## PW-013-07 — 조직 보고서 (SHOULD)

### 자동 보고서

- 주간: 전체 적합화 현황 + 비용 + HITL 처리율
- 월간: 팀별 비교 + 크로스 팀 적합화 갭 + ROI

### 수동 보고서 5종

1. 팀 간 적합화 비교 (선택한 N개 팀)
2. 특정 기간 비용 분석 (모드별)
3. 규칙 적용 현황 (조직 MUST + 팀 규칙 + 프로젝트 규칙)
4. HITL 처리 분석 (평균 대기 / 병목 인물)
5. 사용자 활동 (미사용자 / 활발한 사용자)

### 출력

- PDF (보고 용도, 차트 포함)
- CSV (엑셀 분석)

## PW-013-08 — OA 전용 기능 권한 검증 (MUST)

G-053 정합. 다음 행위는 **OA만** 가능:

- 조직 MUST 규칙 추가/수정/폐기
- 팀 생성/삭제
- 사용자 역할 부여 (OA/L1/L2/L3/L4)
- SSO 설정 변경
- 데이터 보유 기간 변경
- 조직 전체 감사 로그 조회

### 서버 레벨 검증 (G-052)

```typescript
// API 엔드포인트
export const POST = requireLevel('OA', async (req, session) => {
  // 비즈니스 로직
});
```

클라이언트 조건 분기 금지.

## PW-013-09 — 페이지 접근 차단 (MUST)

OA 아닌 사용자가 `/org/*` 직접 접근 시:

- 서버: `403 Forbidden` 반환
- 클라이언트: "이 페이지는 조직 관리자만 접근 가능합니다" 표시 + 대시보드 리다이렉트
- 감사 로그 기록: "권한 거부 발생" (G-141)

## 자동 검증 체크리스트

체인 실행 중 아래 감지 시 Conflict 자동 발동:

- [ ] OA 전용 API가 L1 이하 접근 시 403 반환 안 함?
- [ ] 조직 MUST 규칙이 팀/프로젝트에서 해제 가능 (G-042 위반)?
- [ ] 인프라 모드를 OA가 변경 가능 (G-081 위반)?
- [ ] 조직 감사 로그에 UPDATE/DELETE 버튼 존재?
- [ ] SSO 매핑 없이 수동 역할 할당만 가능 (권장)?
- [ ] 클라이언트 `if (level === 'OA')` 분기 (G-052 위반)?
- [ ] 네비게이션에 OA 아닌데 "조직 관리" 탭 노출?
